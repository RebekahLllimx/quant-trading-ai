## 1. 策略定义与核心概念

### 1.1 策略来源

海龟交易策略（Turtle Trading System）由 Richard Dennis 和 William Eckhardt 于 1983 年提出。Dennis 通过培训 23 名零基础学员（"海龟"），在四年内创造了超过 1 亿美元的利润，证明交易可以被传授。

### 1.2 三层设计哲学

1. **趋势跟踪**：不预测方向，只跟随已形成的趋势。假设趋势比反转更可能延续。
2. **完全机械化**：入场、仓位、止损、离场均由数学规则确定。无主观判断、无情绪干扰。
3. **风险管理优先**：每一笔交易的亏损上限通过波动率（N 值）预先量化——在入场之前就知道最坏结果。

### 1.3 核心概念定义

| 概念 | 严格定义 | 在海龟策略中的角色 |
|------|---------|-------------------|
| **唐奇安通道** | 上轨 = 过去 N 日最高价的滚动最大值；下轨 = 过去 N 日最低价的滚动最小值 | 上轨被突破 = 入场信号；下轨被跌破 = 离场信号 |
| **真实波幅 (TR)** | `TR_t = max(High_t − Low_t, |High_t − Close_{t−1}|, |Low_t − Close_{t−1}|)` | 捕捉日内振幅和隔夜跳空，不遗漏任何形式的跨日价格运动 |
| **N 值 (ATR-20)** | TR 经 Wilder 平滑（α=1/20）后的值，即 20 日 ATR | 海龟策略的基本波动率单位。仓位大小、止损距离、加仓间距均以 N 为单位 |
| **硬止损** | `StopPrice = EntryPrice − k×N_t`（k 默认 2），持仓期间只上移不下移 | 限制单笔交易的最大亏损 |
| **通道离场** | 价格跌破 exit_period 日最低价 | 趋势反转的确认信号，是常规退出的主要机制 |
| **风险预算仓位** | `Shares = RiskAmount / (k × N_t)`，其中 `RiskAmount = Capital × Risk%` | 使每一笔交易在止损触发时的最大亏损恒定为资金的 Risk% |

### 1.4 仓位公式（严格推导）

常见简化写为 `Shares = (Cash × Risk%) / N`。严格推导如下：

```
RiskAmount  = Cash × risk_percent              // 这笔交易允许亏多少钱
StopLoss    = atr_stop_mult × N_t              // 每股止损距离（元）
Shares      = RiskAmount / StopLoss            // 股数 = 允许亏损 / 每股亏损距离
            = (Cash × risk_percent) / (atr_stop_mult × N_t)
```

当 `atr_stop_mult = 2.0` 时，等式为 `Shares = (Cash × risk%) / (2×N)`。简化版的分母是 N，严格版的分母是 2N——差了一倍。**本次实现使用严格版本。**

### 1.5 海龟双系统

| 参数 | System 1（短线） | System 2（长线） |
|------|------------------|-------------------|
| 入场通道周期 | 20 日 | 55 日 |
| 离场通道周期 | 10 日 | 20 日 |
| 止损倍数 | 2×N | 2×N |
| 适用场景 | 趋势明确、波动适中 | 趋势初期、波动较大 |

### 1.6 仅做多

与 Task3 保持一致，本次回测仅实现做多方向。理由：

## 2. 实验设计

> 这是本次 spec 最重要的部分。实验设计的核心目标不是"验证海龟是否有用"，而是回答"**海龟在什么市场条件下有效、在什么条件下失效**"。

### 2.1 标的选取（三类，各有明确的选取逻辑）

不选择单一股票作为主力标的。选取三类标的，每类的选取逻辑可被追问和辩护：

| 类别 | 标的 | 选取逻辑 | 特征（回测前不做预设，仅描述可观测属性） |
|------|------|---------|----------------------------------------|
| 大盘蓝筹 | 贵州茅台 (600519) | Task3 已用双均线测试过茅台，同一标的可直接对比两种策略差异。市值大、流动性好、数据完整 | 日波动率相对低、换手率低 |
| 高成长/新能源龙头 | 宁德时代 (300750) | 创业板权重、新能源产业链核心标的。行业处于成长期，价格波动幅度大 | 日波动率高、Beta 高 |
| 市场指数 | 沪深300 ETF (510300) | ETF 天然代表市场整体走势，无个股基本面干扰。老师不会问"为什么 ETF 代表指数" | 分散化、市场平均波动水平 |

**注意**：不预设宁德时代为"趋势股"。回测后根据 ADX 均值等客观指标判断各标的的实际趋势特征，再用结果讨论海龟策略的适用条件。避免"先预设结论再找证据"。

**数据获取**：510300 需要通过 AKShare 拉取（`ak.stock_zh_a_hist(symbol="510300", period="daily", start_date="...", end_date="...", adjust="qfq")`），存入 `data/csv/510300_沪深300ETF_A股_daily.csv`。

### 2.2 参数对照实验矩阵

固定初始资金 100 万、手续费 0.03%、滑点 0.01%。三组实验：

**实验 A：通道周期对比（固定 ATR=20, 止损=2×N, 标的=茅台）**

| 实验组 | entry | exit | 可测量的预测 |
|--------|-------|------|-------------|
| 短周期 | 10 | 5 | 交易次数 > 中周期的交易次数 |
| 中周期 (System 1) | 20 | 10 | 海龟默认参数 |
| 长周期 (System 2) | 55 | 20 | 交易次数 < 中周期的交易次数；单笔平均收益 > 中周期 |

**实验 B：止损倍数对比（固定 entry=20, exit=10, ATR=20, 标的=茅台）**

| 实验组 | 止损倍数 | 可测量的预测 |
|--------|---------|-------------|
| 紧止损 | 1.5×N | 胜率 < 2.0×N 的胜率；止损退出占比 > 2.0×N |
| 标准止损 | 2.0×N | 海龟默认参数 |
| 宽止损 | 3.0×N | MDD > 2.0×N 的 MDD；止损退出占比 < 1.5×N |

**实验 C：跨标的 × 跨策略对比（固定 System 1 参数）**

不是单独的"跨标的"实验，而是 **4 策略 × 3 标的** 的二维矩阵：

| | 茅台 (600519) | 宁德时代 (300750) | 沪深300 ETF (510300) |
|---|:--:|:--:|:--:|
| 海龟 System 1 (20/10) | 回报 / MDD / 夏普 / 胜率 | 同左 | 同左 |
| 海龟 System 2 (55/20) | 同上 | 同上 | 同上 |
| 双均线 MA5/MA15 (Task3) | 同上 | 同上 | 同上 |
| 买入持有 | 同上 | 同上 | 同上 |

分析时沿两个方向切片：
- **纵向（同标的、不同策略）**：在茅台上海龟 vs 双均线，哪个更好？
- **横向（同策略、不同标的）**：海龟 System 1 在宁德上比在茅台上表现好多少？

### 2.3 可证伪的研究假设

每一个假设都绑定具体的可测量指标和验证条件。如果一个假设被数据推翻，这个"推翻"本身就是有价值的发现。

| # | 假设 | 测量指标 | 验证方式 | 可被推翻的条件 |
|---|------|---------|---------|--------------|
| **H1** | 海龟策略（System 1）的 MDD 低于双均线策略，且该差异在三个标的上一致 | MDD（%） | 计算三个标的上两种策略的 MDD 差值，看方向是否一致 | 如果三个标的中有两个海龟 MDD 反而更高 → H1 被推翻 |
| **H2** | 通道周期越短（10→20→55），交易次数单调递减 | 交易次数（笔） | 实验 A：统计 entry=10/20/55 时的交易次数 | 如果 55 日交易次数反超 20 日 → H2 被推翻 |
| **H3** | 止损倍数越大（1.5→2.0→3.0），最大回撤单调递增 | MDD（%） | 实验 B：比较 1.5/2.0/3.0×N 的 MDD 值 | 如果 3.0×N 的 MDD 反而小于 1.5×N → H3 被推翻 |
| **H4** | 海龟策略在高波动标的（宁德时代）上的累计回报绝对值 > 低波动标的（茅台），但夏普比率不一定更高（因为波动大） | 累计回报（%）、夏普比率 | 实验 C：比较宁德 vs 茅台的策略累计回报和夏普 | 如果茅台的策略回报 > 宁德 → H4 被推翻；如果宁德的夏普也更高 → H4 部分被推翻 |

## 3. 数据与指标计算

### 3.1 数据来源

- 使用 `data/csv/` 下已有 CSV 文件（AKShare 前复权 qfq 日线数据）
- 字段：Date, Open, High, Low, Close, Volume
- 无需重新拉取

### 3.2 唐奇安通道

```
DC_High(N)_t = max(High_{t−N+1}, ..., High_t)
DC_Low(N)_t  = min(Low_{t−N+1}, ..., Low_t)
```

`pandas` 实现：`high.rolling(window=N, min_periods=N).max()`。

入场通道周期 `entry_period`，离场通道周期 `exit_period`。两者独立计算，分别控制入场和离场的时间尺度。

### 3.3 N 值 (ATR)

```
TR_t = max(High_t − Low_t, |High_t − Close_{t−1}|, |Low_t − Close_{t−1}|)

N_t = WilderSmooth(TR, period)_t
    = N_{t−1} × (period−1)/period + TR_t × (1/period)
```

- **Wilder 平滑 α = 1/period**（非 EMA 的 α = 2/(period+1)）
- 种子值 = TR 的前 period 期 SMA
- 复用 `src/indicators.py` 中的 `calc_atr()` 函数

### 3.4 避免未来函数

- 入场：T 日盘中最高价 > T-1 日入场通道上轨 → T 日收盘价执行交易
  - T-1 通道值在 T 日开盘前已知 ✓
  - 等价于海龟的 Stop Order（在 N 日高点上方挂突破买入单）
- 离场：T 日最低价 < T-1 日离场通道下轨 → T 日收盘价执行
- 止损：T 日最低价 ≤ 动态止损价 → 以止损价卖出

## 4. 回测引擎算法

### 4.1 参数表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `entry_period` | 20 | 入场通道周期（日） |
| `exit_period` | 10 | 离场通道周期（日） |
| `atr_period` | 20 | N 值计算周期（日） |
| `atr_stop_mult` | 2.0 | 止损距离 = k×N |
| `risk_percent` | 0.02 | 单笔风险预算占资金比例 |
| `initial_capital` | 1,000,000 | 初始资金 |
| `fee_rate` | 0.0003 | 手续费率 |
| `slippage` | 0.0001 | 滑点率 |
| `use_filter` | True | 是否启用亏损过滤（上一笔亏损则跳过下次 System 1 信号） |

### 4.2 伪码

```
function turtle_backtest(df, ...):

    // 1. 计算指标
    N  = WilderSmooth(TR, atr_period)
    DC_entry = Donchian(High, Low, entry_period)
    DC_exit  = Donchian(High, Low, exit_period)

    // 2. 初始化
    cash = initial_capital; shares = 0; has_position = false
    stop_price = 0; last_trade_lost = false

    // 3. 逐日遍历
    start_idx = max(entry_period, exit_period, atr_period) + 2
    for t = start_idx to len(df)-1:

        equity = cash + shares × Close[t]
        record equity_curve

        if not has_position:
            // 入场：T日High突破T-1日通道上轨
            if High[t] > DC_entry.High[t-1]:
                if use_filter and last_trade_lost: skip
                risk_amt = cash × risk_percent
                stop_dist = atr_stop_mult × N[t]
                shares = risk_amt / stop_dist              // ★ 严格公式
                shares = min(shares, cash / (buy_price × (1+fee_rate)))
                buy_price = Close[t] × (1 + slippage)
                cost = shares × buy_price × (1 + fee_rate)
                cash -= cost
                entry_price = buy_price
                stop_price = entry_price - stop_dist
                has_position = true
                record BUY trade

        else:
            // 动态止损上移（只上移不下移）
            trail_stop = Close[t] - atr_stop_mult × N[t]
            if trail_stop > stop_price: stop_price = trail_stop

            // 检查止损（优先级高于通道离场）
            if Low[t] <= stop_price:
                sell at stop_price × (1-slippage)
                last_trade_lost = (return < 0)
                record SELL (exit_reason: 止损)

            // 检查通道离场
            else if Low[t] < DC_exit.Low[t-1]:
                sell at Close[t] × (1-slippage)
                last_trade_lost = (return < 0)
                record SELL (exit_reason: 通道离场)

    // 4. 强制平仓
    if has_position: sell at Close[-1]

    // 5. 基线计算
    bh_equity = buy_and_hold(df, initial_capital)

    return {trades, equity_curve, bh_equity, signals, params}
```

### 4.3 关键设计决策

| 决策点 | 选择 | 依据 |
|--------|------|------|
| 入场触发 | T 日 High > T-1 日通道上轨 | 等价海龟原版 Stop Order |
| 仓位公式 | `RiskAmt / (k×N)` | 严格推导，使止损触发时亏损恰好 = RiskAmt |
| 止损触发价 | 止损价 × (1−slippage) | 保守假设：止损单可能无法精确成交 |
| 止损 vs 离场优先级 | 先止损、后离场 | 止损 = 紧急退出，离场 = 常规退出 |
| 动态止损方向 | 只上移不下移 | 多头持仓止损价不应降低 |
| 做空 | 不实现 | A 股融券受限，与 Task3 可对比 |

## 5. 评估指标与基线对比

### 5.1 核心指标表

| 指标 | 公式 | 为什么重要 |
|------|------|-----------|
| 累计回报 | `(V_final − V_init) / V_init × 100%` | 总收益，最直观 |
| 年化收益率 | `(V_final/V_init)^(252/days) − 1` | 可跨时间段比较 |
| **最大回撤 MDD** | `min((Equity_t − Peak_t) / Peak_t)` | ★ 海龟最重要的风险指标——连续亏损期的最大跌幅直接影响交易者能否坚持执行 |
| 夏普比率 | `sqrt(252) × (μ − Rf/252) / σ` | 风险调整后收益，Rf=2% |
| 胜率 | `N_win / N_total × 100%` | 海龟典型胜率 30-40%，越低越需要盈亏比支撑 |
| 盈亏比 | `Avg(win_return) / Avg(|loss_return|)` | 低胜率必须配高盈亏比才能盈利 |
| 年化波动率 | `σ_daily × sqrt(252)` | 策略资产本身的波动水平 |
| 交易次数 | 完整买卖周期数 | 判断是否过度交易 |
| 止损退出 vs 信号退出 | 分别统计 | 判断哪种退出机制主导——如果全部是止损退出，说明通道离场设置可能太宽松 |

### 5.2 基线对比矩阵

每只标的回测结果同时报告三列：

| 列 | 含义 |
|----|------|
| 海龟策略 | 本次实现的回测结果 |
| 买入持有 | 同期买入持有，作为"什么都不做"的基线 |
| 双均线 (Task3) | MA5+MA15 交叉策略，作为"另一种趋势跟踪策略"的基线 |
