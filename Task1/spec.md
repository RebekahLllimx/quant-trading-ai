## 2. 术语定义

### 2.1 K线（Candlestick / K-Line）

一根 K 线包含四个价格要素：

| 要素 | 英文 | 含义 |
|------|------|------|
| 开盘价 | Open | 该周期第一笔成交价 |
| 收盘价 | Close | 该周期最后一笔成交价 |
| 最高价 | High | 该周期最高成交价 |
| 最低价 | Low | 该周期最低成交价 |

**实体**: 开盘价与收盘价之间的矩形区域。收盘 > 开盘为阳线（红色/空心），收盘 < 开盘为阴线（绿色/实心）。

**影线**: 实体上方为上影线（顶端 = 最高价），实体下方为下影线（底端 = 最低价）。

K 线起源于 18 世纪日本大米期货市场（本间宗久），是目前全球金融市场最通用的价格图表形式。常见周期：日 K、周 K、月 K，以及 5min/15min/60min 等分钟级 K 线。

### 2.2 基本面分析（Fundamental Analysis）

通过评估内在价值判断证券是否被高估或低估的方法论。三层架构：

- **宏观层**: GDP、通胀、利率、货币政策、就业数据
- **行业层**: 生命周期、竞争格局、监管政策、技术变革
- **公司层**: 财务报表（资产负债表/利润表/现金流量表）、PE、PB、ROE、自由现金流

代表人物：格雷厄姆（价值投资之父）、巴菲特。适合中长期投资，局限是信息获取和分析耗时长，且市场可能长期偏离基本面。

### 2.3 技术分析（Technical Analysis）

通过研究市场交易数据（价格、成交量）预测未来走势的方法。三个基本假设：

1. 市场行为包容消化一切信息
2. 价格以趋势方式演变（惯性）
3. 历史会重演（人性不变）

主要工具：图表形态（头肩顶/底、双重顶/底、三角形等）、技术指标（MA、MACD、RSI、KDJ 等）、道氏理论/波浪理论/江恩理论。

技术与基本面并非互斥——常见的分工是：基本面选股（买什么），技术择时（何时买卖）。

## 3. 数据源选型

### 3.1 双源架构

| 数据源 | 角色 | 特点 |
|--------|------|------|
| **AKShare** | 主力 | 免费、无限制、覆盖 A 股+港股、接口简洁 |
| **Tushare Pro** | 补充 | 需注册 Token、港股限流（1次/小时）、数据质量高 |

**决策**: 主推 AKShare，Tushare 作为 fallback。原因：AKShare 无需注册即可使用，对港股无频率限制，更符合课程场景下快速迭代的需求。

### 3.2 复权方式

统一使用 **前复权（qfq）**。前复权以当前股价为基准向前调整历史价格，适合技术分析——当前价格是真实的，历史价格是调整后的，不影响最新 K 线的形态判断。

## 4. 标的选取

### 4.1 样本股票

选取 **平安银行（000001.SZ）** 作为核心示例，原因：
- 深交所首只股票，代码辨识度高
- 银行股波动相对平稳，适合入门演示
- 流动性充裕，数据完整

## 5. 架构设计

### 5.1 数据流

```
AKShare / Tushare API
        │
        ▼
  update_data.py          ← 数据获取 + 收盘价图 + CSV
        │
        ├──► data/csv/{code}_{name}_{market}_daily.csv
        │
        ▼
  plot_candlestick.py     ← A股+港股 K 线图（mplfinance）
  plot_multi_stock.py     ← 5 只 A 股对比分析（mplfinance）
        │
        ├──► data/charts/task1/*.png
        │
        ▼
  build_dashboard.py      ← CSV → JSON 嵌入 → 自包含 HTML
        │
        ▼
  dashboard/index.html    ← 交互看板（ECharts 5.5）
        │
        ▼
  generate_report.py      ← 报告生成（python-docx）
        │
        ▼
  Rebecca+Task1.docx
```

### 5.2 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 复权 | 前复权 (qfq) | 适合技术分析，当前价格真实 |
| 绘图库 | mplfinance | 专业金融 K 线图，多面板支持，中文兼容 |
| 看板 | ECharts 5.5 自包含 HTML | 无需后端部署，GitHub Pages 兼容 |
| 报告 | python-docx | CJK 字体精确控制，程序化生成 |
| CSV 编码 | utf-8-sig | Excel 直接打开不乱码 |
| 图表路径 | data/charts/task1/ | 与 Task2 等共享 data/ 根目录结构 |

## 6. 看板方案

### 6.1 布局

- **顶部导航栏**: 深色渐变背景，项目名 + GitHub 链接
- **左侧边栏** (300px): 标的列表（A 股 / 港股分组），点击切换
- **右侧图表区**: 上下两图——K 线图（含 MA 均线 + 成交量）+ 数据分析（日收益分布 + 累计收益）

### 6.2 交互功能

- 10 只标的可点击切换
- K 线图支持缩放、拖拽、十字光标
- 日收益直方图 + 正态拟合曲线
- 累计收益曲线
- 下载当前标的 OHLC 数据为 CSV

### 6.3 视觉规范

- 主题: 浅色（`--bg:#f5f6fa`, `--card:#fff`, `--accent:#0984e3`）
- K 线配色: 涨红 `#ef5350`，跌绿 `#26a69a`
- 图表标题: 居中 `{标的名称} ({代码})`
